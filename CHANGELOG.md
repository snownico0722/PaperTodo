### 评估

- **Scoop / Choco 软件源**：Scoop 更适合当前绿色免安装形态，能让安装和升级更轻；Chocolatey 也可考虑，但更偏系统包管理。需要额外维护发布包地址、校验值和版本清单，并确认用户数据、自定义图标等仍保留在程序目录或约定位置，短期可先记录为发布渠道评估项。
- 单独调整胶囊缩放
- 单独调整整体视觉大小（和windows显示比例一个逻辑）
- 在win 10多个桌面（如下图）之间的窗口管理也许可以优化？比如如果在桌面1点击胶囊，对应便签在桌面2，系统会让我切到桌面2，我需要手动把便签从桌面2移动到桌面1；我希望在桌面1点击胶囊，胶囊对应的便签直接在桌面1打开（桌面2的便签窗口可以关闭掉）
-  窗口系留模式
  目前纸片只能常驻于桌面或置顶于所有窗口之上。但在实际工作流中，便签往往是服务于某个特定具体任务的（比如对着一段产品文档写代码，或者看着参考图作画）。
  - **全新交互**：新增“系留（Anchor）”功能。你可以把一张纸片像吸铁石一样，“啪”地一下贴到任意一个第三方软件窗口的边缘（比如贴在 Photoshop、Unity、VS Code 或浏览器的右侧）。
  - **工作流联动**：一旦完成系留，这张纸片将动态跟随该目标软件。目标软件最小化，纸片自动化为胶囊或隐藏；目标软件切到前台，纸片同步在它的边缘滑出。
  - **技术实现**：利用 Win32 的 `SetWinEventHook` 监听目标窗口的句柄（HWND）状态（移动、最小化、激活），在 WPF 层实现无感知的坐标联动，真正做到“如影随形”却不干扰全局桌面。
- 幽灵壁纸模式绝对不占层级的无感存在置顶胶囊有时候还是会挡住一些极限操作边缘，而非置顶形态又容易被大窗口彻底埋没。我们需要一种介于两者之间的“幽灵状态”。  全新交互：开启后，纸片会“半透明融化”进你的桌面壁纸里。它不占用任何合法的窗口层级，平时完全鼠标穿透（Click-through），你拖动桌面图标或框选文件时，它就像纹身一样印在壁纸上。唤醒机制：当你按住一个自定义全局修饰键（比如长按 Caps Lock 或连续晃动鼠标），幽灵纸片瞬间恢复全实体并获取输入焦点，松开即再次隐形。技术实现：通过修改窗口扩展样式 WS_EX_TRANSPARENT（鼠标穿透）和调整 DWM 的恶魔级 Z-order（将窗口层级强行插在系统 Progman / WorkerW 壁纸层之上），实现极致纯净的原生桌面看板效果。
- 普通胶囊磁吸吸附：评估为非侧边队列的普通胶囊增加轻量磁吸能力，拖动靠近窗口或屏幕边缘时可吸附，并在目标移动时尝试跟随；不影响侧边胶囊队列、主胶囊和纸片展开逻辑，拖离或目标失效后自动解除吸附。

- 视觉页新增图片标记显示方式，可选择“始终显示 / 仅编辑时 / 始终隐藏”；隐藏只影响界面，不会删除或修改 Markdown 原文。
- 视觉页新增「自定义字体增强加粗」：程序目录同时有 `papertodo` 正文字体与粗体文件（如 `PaperTodo_Bold.ttf` / `papertodo_bold.ttf`）时可勾选；开启后，笔记、待办、标题和胶囊中已勾选加粗的文字会改用该粗体文件显示，而不是系统合成半粗（部分可变字体或仅含 Regular 的自定义字在合成加粗时几乎看不出效果）。

**胶囊快捷操作与快捷键**

- 胶囊模式下，无论纸片处于编辑态还是浏览态，按一次 Escape 即可提交当前编辑内容并收回胶囊。
- 快捷键页支持为显示全部、隐藏全部、切换显隐、新建待办、新建笔记、退出录入全局快捷键；名称更简短直接，录入已经注册的组合键时会将其作为新输入，不会误执行原命令。
- 每项全局快捷键均可独立启用；录入、恢复默认、开关切换都会立即生效并自动保存，不再需要单独点保存。录入组合键时会自动勾选启用；单项或整表恢复默认时按默认启用状态写入（边缘序列默认关闭）。新增默认关闭的「快速启动侧边胶囊」：左侧队列默认 `Ctrl+Shift`、右侧队列默认 `Ctrl+Alt`，每侧一行设置；同时按下并松开 2～3 个修饰键即可录入，数字 1–9 固定显示且不可改，左右不能使用相同修饰键，开关一次启用本侧整组。调用时优先查找鼠标所在显示器的指定侧与对侧，再按左邻、右邻顺序搜索其他显示器，主胶囊不计入序号；拖动任意边缘胶囊期间会忽略这些快捷键。
- 应用快捷键前会同时检查应用内重复和系统占用，任一组合不可用时立即恢复并显示原有配置，同时保留本次失败提示。
- 可限制边缘胶囊标题最多显示前 N 个 Unicode 字符，默认“全部”兼容原有宽度；普通胶囊会在字体变化及形态过渡收尾时重新校正宽度，贴近右侧工作区时保持右边缘不动。

**笔记图片**

- 笔记支持从剪贴板、拖入或菜单插入图片，按纸片宽度完整显示；选中图片后可通过删除对应 Markdown 引用。发布包已内置图片存储支持，无需用户另行安装或配置组件；混合剪贴板优先粘贴文字，剪贴板内容暂时无法读取时只取消本次粘贴，不再导致程序退出。
- 新插入图片默认使用 `![image|100%](i:001)` 短编号语法，支持 `![image|宽x高, 75%](...)` 或 `{width=75%}` 调整显示大小；复制、粘贴和撤销不会夹带隐藏字符，把图片引用粘贴到另一笔记时会复制对应图片数据，不再依赖原笔记。
- 图片以增量方式存储在单个本地资产文件中，不会因图片增多而反复重写全部数据；单张图片元数据或内容损坏时只隔离该图片并在原位置显示损坏状态，不影响其他图片继续读取和写入；删除或剪切图片后仍可正常撤销，当前数据、备份和恢复副本都不再需要的图片才会被安全清理；外部打开笔记时会安全导出临时文件。
- 新增默认开启的“自动压缩过大图片”设置：超过 8 MB 或长边超过 2048 像素时尝试等比压缩；压缩不安全、失败、仍超限或文件没有变小时取消导入，宽或高超过 4096 像素时直接拒绝。关闭后保留原图，超过 8 MB 时直接拒绝。
- 图片导入失败和不支持的拖入格式会通过纸片样式弹窗明确提示，不再静默忽略。
- 批量插入图片会先确认正文容量，再以一笔资产事务写入并一次更新正文；任一图片导入失败时正文保持不变。
- 过大的源图片文件在读取前会被拒绝；外部打开时如临时图片无法完整导出会提示并取消打开。

**胶囊相关改动**

- 拖动边缘胶囊排序或跨队列移动时，当前胶囊始终保持在最上方不被遮挡。
- 收紧边缘胶囊悬停时的关闭区和伸出长度，进一步减少主胶囊右侧留白并加快胶囊滑出；纸片收起后边缘胶囊会平滑缩回，拖动排序经过其他胶囊时，其他胶囊不再被误触发伸长。
- 关闭胶囊模式时，已折叠纸片正常展开为纸片形态
- 边缘胶囊与 slot 0 主胶囊改为按实际显示宽度绘制单侧圆角贴边标签；胶囊本体始终钉住屏幕墙边并只向屏幕内部伸缩，关闭区固定在墙边随本体展开，拖出队列时改用不含关闭区的完整悬浮胶囊，不再通过裁切、屏幕外溢出或复用悬浮布局形成外观。
- 贴边胶囊的纸片类型图标调整到标题区域靠屏幕内侧的一端，左右边缘保持镜像布局。
- 优化多个边缘胶囊同时让位、排序或收向主胶囊时的动画开销，减少队列较长时的卡顿。

- 优化冷启动和多纸片恢复速度，更快显示纸片。

- 修复最小化的纸片无法恢复显示的问题。

- 拖动排序期间发生的队列重排和标题宽度刷新会在拖动结束后统一应用；伸缩中的标题变化会平滑接续，不再丢失刷新或硬切关闭区。
- 边缘胶囊关闭区恢复到屏幕墙边；左右胶囊都按真实可交互区域判断悬停，移开后会稳定收回，右侧伸缩不再先离开墙边露出缝隙或因窗口移动反复闪烁。
- 修复关联笔记改绑、删除空待办、Undo/Redo 后胶囊显隐未及时更新的问题；多行粘贴待办只需一次撤销即可完整恢复。
- 修复从待办打开已折叠的关联笔记时，胶囊可能先在历史位置闪现再跳到待办旁的问题。
- 优化 Windows 分屏贴靠显示：贴靠时自动隐藏纸片阴影和边距，识别三列中间格、2/3 主列等 Windows 11 布局。
- 分屏纸片折叠为胶囊后视为普通纸片，再次展开恢复为分屏前的大小；隐藏再显示不会跳回旧分屏格子。
- 笔记达到 100000 字符后不再后台截掉已有正文；继续输入会被阻止，超过单次 30000 字符、单行 6000 字符或笔记总上限的粘贴和普通文本拖放会整次拒绝并给出一次提示。
- 退出时可靠回收本次运行启动的全部 PowerShell 进程（包括仍在执行的一次性脚本），单个窗口或清理步骤异常不会阻断完整退出。
- 退出时若最终保存失败，会在关闭前明确提示本次未保存的修改可能丢失；仍在设置页编辑的外部 Markdown 文件后缀也会先纳入最终保存。
- 修复纸片缩放到最小尺寸附近时可能越过限制并错误裁切内容的问题。

### v0.1 初始构建

- **全新原生框架**：基于 .NET 10 和 WPF 打造的轻量级无主窗口“纸片”待办应用，绿色免安装。
- **无重度依赖**：不使用 WebView、Tauri、Electron 等重度渲染引擎，无 MSIX/Store/AppX 打包限制，系统资源占用极低。
- **本地数据优先**：支持基础的待办管理与便签创建，数据采用本地 JSON 格式存储，完全离线。

### v0.2 纸片生命周期

- **多纸片管理**：支持创建和管理多张独立的纸片窗口，自动记录每张纸片的位置、大小、置顶状态与内容。
- **启动智能恢复**：程序启动时自动恢复此前未删除的纸片；检测到纸片因多显示器切换等原因超出当前屏幕边界时，会自动拉回可视区域，防止窗口丢失。
- **数据安全备份**：引入 data.backup.json 自动备份与损坏恢复机制，在主配置文件意外受损时可尝试自动恢复。
- **生命周期规范**：关闭窗口即为隐藏，点击删除才会触发真正的确认移除。

### v0.3 待办纸基础交互

- **待办快捷操作**：支持通过 Enter 快速新建待办、Backspace 删除空行，勾选复选框时已完成项会自动添加删除线并淡化显示。
- **追加与清理**：提供底部 ＋ 快速追加区，支持一键清除所有已完成事项，也可以单独删除单项。
- **输入节奏优化**：优化空行限制规则，允许连续创建并保留多个空待办行。

### v0.4 托盘与纸片入口

- **托盘中心控制**：引入任务栏托盘图标，支持一键新建待办/笔记、显示/隐藏全部纸片以及快速退出。
- **托盘纸片列表**：托盘菜单可实时列出所有活跃纸片并支持长标题自动省略，点击对应的纸片名称可快速切换显隐。
- **预览命名与置顶**：纸片名称可根据第一行内容自动生成；纸片左上角功能图标改置顶开关。
- **自定图标支持**：程序会自动读取同级目录下的 PaperTodo.ico 替换默认托盘图标。

### v0.5 菜单与视觉整理

- **右键菜单优化**：右键菜单重构为四个分类清晰的分组（事项、格式、编辑、当前纸片），移除了传统 Windows 右键菜单左侧的空白图标预留区。
